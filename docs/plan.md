# Dranimo 手绘动画 MVP 实施计划

## 总体方案

- 构建本地优先的单页手绘动画工具，支持鼠标、触控板和触摸输入。
- 使用 `perfect-freehand` 生成笔画轮廓，使用 `localforage` 自动持久化单个项目。
- 使用 `mediabunny` 统一处理视频编码、WebM 封装、视频能力检测和透明视频验证，不直接调用 `MediaRecorder`。
- 支持动画预览以及 PNG、SVG、WebM 导出；WebM 首版明确支持桌面 Chrome/Edge。

## 画板与动画

- 提供画笔、整笔橡皮擦、撤销和重做；一次橡皮擦拖动产生一个可撤销操作。
- 基础画笔设置包含颜色、粗细和透明度；高级设置默认折叠，包含 `thinning`、`smoothing`、`streamline`、`simulatePressure` 和 taper/cap。
- 画笔设置只影响新笔画；首版不支持选择、移动或修改已有笔画。
- 提供 1:1（1080×1080）、16:9（1920×1080）、9:16（1080×1920）画布和纯色/透明背景。
- 编辑区域响应式缩放，所有输入转换为画布逻辑坐标；切换比例不缩放或删除已有笔画。
- 动画按笔画创建顺序回放，提供播放、暂停、重播和进度拖动。
- 支持三种节奏模式：
  - 真实速度：使用采样时间还原单条笔画速度，忽略笔画之间的原始停顿。
  - 固定速度：根据路径长度和逻辑像素/秒计算时长，默认 600px/s。
  - 总时长：扣除笔画间隔后，按路径长度比例分配时长，默认总时长 5 秒。
- 所有模式使用统一笔画间隔，默认 120ms；点状笔画以笔刷尺寸作为长度权重。
- 总时长不足以容纳笔画间隔时阻止播放和导出，并显示调整提示。
- 预览与视频导出共用 `buildPlaybackSchedule` 和 `renderFrame(time)`。

## 数据与持久化接口

- 新增运行时依赖：`perfect-freehand`、`localforage`、`mediabunny`。
- 编辑器实现为 Client Component；页面壳与 Metadata 保持 Next.js 服务端组件。
- 建立版本化数据模型：
  - `StrokePoint`：逻辑坐标、压力、相对时间。
  - `StrokeRecord`：ID、采样点、颜色、透明度和完整画笔参数快照。
  - `CanvasSettings`：比例、逻辑尺寸和编辑器背景色；编辑器始终显示纯色背景，旧项目的背景模式字段仅作兼容保留。
  - `PlaybackSettings`：模式、笔画间隔、固定速度和目标总时长。
  - `ProjectV1`：`version: 1`、笔画、画布、动画和当前画笔设置。
  - `ExportSettings`：格式、导出背景模式、完整画布/自适应、倍率、留白和帧率。
- 使用 localForage 的独立 `dranimo` store 保存唯一项目。
- 初始化加载和版本校验完成后才启动自动保存，避免默认状态覆盖已有数据。
- 笔画和擦除完成后立即保存；设置变更采用 300ms 防抖。
- 刷新后恢复项目，但不保存撤销/重做历史。
- 数据损坏或读取失败时显示错误并进入默认空画布，不自动覆盖损坏数据。

## 图片与视频导出

### 通用范围

- 支持完整画布和自适应裁切。
- 自适应模式根据最终所有笔画轮廓的联合边界裁切，默认添加 32 个逻辑像素留白并允许调整。
- 自适应视频全程使用最终联合边界，不随动画进度移动或缩放镜头。
- 提供 1×、2×、3×倍率，默认 2×；最长边超过 3840px 时等比缩小，并提前显示实际导出尺寸。
- 空画布禁用导出。

### PNG 与 SVG

- PNG 使用离屏 Canvas 渲染；透明背景保留 Alpha。
- SVG 将 perfect-freehand 轮廓输出为闭合路径。
- 导出时可选择透明或纯色背景；透明 SVG 不写背景矩形，纯色背景写入覆盖导出区域的背景矩形。
- PNG、SVG 与动画预览共用轮廓和坐标转换逻辑。

### MediaBunny WebM

- 创建离屏 Canvas，并按动画排程以固定 30fps 调用 `renderFrame(time)`。
- 使用 MediaBunny 的 `Output`、WebM 输出格式、Canvas 视频源和内存输出目标进行离线逐帧编码。
- 优先使用 VP9；通过 MediaBunny 的编码能力检测确认当前浏览器、尺寸和编码配置可用后才开始导出。
- 使用显式时间戳和帧时长写入视频源，不使用 `captureStream()` 或 `MediaRecorder`，因此导出无需按作品时长实时等待。
- 编码完成后由 MediaBunny finalize WebM，转换为 Blob 并触发下载。
- 使用 MediaBunny 的高质量编码配置；首版最大动画时长为 60 秒、最长边为 3840px。
- 导出期间展示帧进度和编码状态；取消时停止帧循环、终止 MediaBunny 输出并丢弃未完成文件。
- 非透明视频先在离屏 Canvas 绘制用户选择的背景色。

### 透明 WebM

- 透明视频仅使用支持 Alpha 的 VP9 WebM 配置，不允许自动铺底或切换成不透明编码。
- 首次透明导出前执行实际能力探测：
  1. 使用 MediaBunny 将带透明像素的测试 Canvas 编码为短 WebM。
  2. 使用 MediaBunny 重新读取并解码该测试文件。
  3. 将解码帧绘制到检测 Canvas，验证目标像素的 Alpha 值。
- 探测结果按浏览器与编码配置缓存到当前会话。
- 探测失败时禁用透明 WebM，并显示“不支持透明视频导出”；PNG 和 SVG 透明导出仍然可用。
- 视频编码、封装、读取和透明验证全部通过 MediaBunny 完成，不自行拼装容器或直接调用底层 WebCodecs。

## 测试与验收

- 单元测试覆盖轮廓生成、整笔命中删除、联合边界、留白、倍率限制和坐标转换。
- 动画测试覆盖三种节奏模式、笔画间隔、点状笔画、非法总时长和任意时间点的局部笔画渲染。
- 状态测试覆盖绘制、擦除、撤销/重做、画笔参数快照及切换画布比例。
- 持久化测试覆盖首次加载、自动保存、防抖、损坏数据及版本迁移入口。
- PNG/SVG 测试覆盖尺寸、裁切、纯色背景和 Alpha。
- MediaBunny 视频测试覆盖：
  - VP9 能力检测和不支持状态。
  - 帧时间戳、视频时长、首帧和最终帧。
  - 完整画布与自适应裁切。
  - 导出进度、取消及未完成输出清理。
  - 透明 WebM 编码—解码—Alpha 像素验证。
- 使用桌面 Chrome/Edge 完成 WebM 端到端验收；其他现代浏览器验收绘制及 PNG/SVG 导出。
- 最终运行 TypeScript、Biome、Next.js production build 和浏览器端测试，并遵循仓库当前 Next.js 16 内置文档。

## 默认范围

- 首版不包含账号、云同步、多作品列表、协作、文字、形状、图片、图层、选择变换、音频或可编辑时间轴。
- 不包含项目 JSON 导入导出、MP4、GIF、服务端渲染或服务端转码。
- 不直接使用 `MediaRecorder`；所有视频相关处理统一由 MediaBunny 承担。
